# Contributing to Kiddlr 🎨

First off, thank you for considering contributing to Kiddlr! It's people like you that make Kiddlr a magical experience for kids around the world.

### 🚀 How to Contribute

1. **Fork the Repository**
   - Click the "Fork" button at the top right of the [Kiddlr repository](https://github.com/vennamprasad/kidsTypo).
   - Clone your fork to your local machine:
     ```bash
     git clone https://github.com/YOUR_USERNAME/kidsTypo.git
     ```

2. **Create a Branch**
   - Create a feature branch for your changes:
     ```bash
     git checkout -b feature/amazing-new-game
     ```

3. **Make Your Changes**
   - Implement your feature or fix.
   - Ensure the code follows the existing style (Tailwind CSS, TypeScript).
   - Test your changes locally: `npm run dev`.

4. **Commit Your Changes**
   - Write a clear and concise commit message:
     ```bash
     git commit -m "feat: add super sparkle effect to magic canvas"
     ```

5. **Push to GitHub**
   - Push your branch to your fork:
     ```bash
     git push origin feature/amazing-new-game
     ```

6. **Submit a Pull Request**
   - Go to the original Kiddlr repository and click "New Pull Request".
   - Describe your changes and why they are beneficial.

### 🧪 Development Guidelines
- **Monorepo**: We use Turborepo. Run commands from the root.
- **Next.js**: The main app is in `apps/web`.
- **Styling**: We use Tailwind CSS v4.
- **State**: Zustand is used for global app state.

### 🤖 AI-First Development
Kiddlr was developed in collaboration with the **Antigravity AI Agent**. We welcome both human and AI-assisted contributions! If you used an AI agent to help with your PR, feel free to mention it in the description.

Happy coding! 🐣
